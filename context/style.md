# 写作风格与文章结构规则

## 一、对话式文章结构

### 1. 开头部分（主持人旁白）
- 整体工具介绍：火爆程度 + 具体功能 + 哪个功能特别值得关注
- 描述格式：先说今天要讲什么 → 详细介绍工具 → 点明值得关注的点

### 2. 中间问答部分（7个问题）
- 问题1：基础功能类（和同类工具的区别）
- 问题2：使用场景类（设计师能用它干什么）
- 问题3：竞品对比类（和 Copilot、Cursor 比）
- 问题4：安全性类（数据在不在云端）
- 问题5：风险评估类（靠谱不靠谱）
- 问题6：职业影响类（设计师会不会被取代）
- 问题7：决策建议类（到底要不要用）

### 3. 结尾部分（小结视角）
- 非对话形式的总结
- 小编视角的表达：「今天给大家介绍的是...」「总的来说...」「好了，今天的分享就到这里」

### 4. 参考来源
- 放在文章最后
- HTML显示，TTS不朗读

---

## 二、各角色说话方式

| 角色 | 出现位置 | 说话方式 |
|------|----------|----------|
| **主持人旁白** | 开头、工具介绍 | 整体介绍、描述火爆程度 |
| **主持人提问** | 每个问题结尾 | 选2个加「你怎么看？」，选1个加「你觉得呢？」，其余不加 |
| **主持人提问** | 每个问题开头 | 「第一个问题是XXX」「比如说XXX」「站在普通用户视角上，你怎么看？」 |
| **嘉宾回答** | 问题回答 | 「这是第一个关键点」「除此以外」「虽然...但是...」「在我看来」「我希望」 |
| **小结视角** | 结尾总结 | 「今天给大家介绍的是...」「总的来说...」 |

---

## 三、回答结构化关键词

| 关键词 | 用途 | 示例 |
|--------|------|------|
| 第一个问题/第一个点 | 引入第一点 | 「这是第一个关键点」 |
| 除此以外 | 补充第二点 | 「除此以外，还有...」 |
| 虽然...但是... | 转折对比 | 「虽然功能有限，但是...」 |
| 在我看来/我个人认为 | 表达观点 | 「在我个人看来...」 |
| 我希望 | 建议/呼吁 | 「我希望你能...」 |
| 比如说/我给你举例子 | 举例说明 | 「我给你举几个例子」 |
| 第一个场景是XXX | 场景描述 | 「第二个场景是XXX」 |
| 站在普通用户视角... | 判断类问题 | 「站在普通用户的视角上，你怎么判断」 |

## 四、内容清洗规则（TTS不读）

| 内容 | HTML显示 | TTS朗读 |
|------|----------|---------|
| 大标题 `#` | ✅ | ❌ |
| 开头背景介绍 | ✅ | ✅ |
| 正文对话内容 | ✅ | ✅ |
| 参考来源列表 | ✅ | ❌ |
| 日期标注 (截至...) | ❌ | ❌ |
| 主持人介绍 | ❌ | ❌ |
| Markdown加粗 ** | 显示 | ❌ |
