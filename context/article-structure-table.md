## 角色说明

| 角色 | 出现位置 | 说话方式 |
|------|----------|----------|
| **主持人旁白** | 开头、工具基础介绍 | 整体介绍、描述火爆程度、功能介绍 |
| **主持人提问** | 每个问题结尾 | 选2个加「你怎么看？」，选1个加「你觉得呢？」，其余不加 |
| **主持人提问** | 每个问题开头 | 「第一个问题是XXX」「比如说XXX」「站在普通用户视角...」 |
| **嘉宾回答** | 问题回答 | 「这是第一个关键点」「除此以外」「虽然...但是...」「在我看来」「我希望」 |
| **小结视角** | 结尾总结 | 「今天给大家介绍的是...」「总的来说...」「好了，今天的分享就到这里」 |

## 回答结构化关键词

| 关键词 | 用途 | 示例 |
|--------|------|------|
| 第一个问题/第一个点 | 引入第一点 | 「这是第一个关键点」 |
| 除此以外 | 补充第二点 | 「除此以外，还有...」 |
| 虽然...但是... | 转折对比 | 「虽然功能有限，但是...」 |
| 在我看来/我个人认为 | 表达观点 | 「在我个人看来...」 |
| 我希望 | 建议/呼吁 | 「我希望你能...」 |
| 比如说/我给你举例子 | 举例说明 | 「我给你举几个例子」 |

## 内容长度

- 当前版本：7个问题 + 开头 + 工具介绍 + 小编总结 + 参考来源
- 预计长度：约1.5倍之前的内容

## 关键规则

1. 开头必须是主持人整体介绍（火爆程度+功能+值得关注的点）
2. 每个问题开头用「第一个问题是XXX」「第二个场景是XXX」
3. 判断类问题加「站在普通用户视角上，你怎么看？」
4. 结尾必须是「小结」视角总结（非对话形式）
5. 参考来源放在最后，HTML显示但TTS不读
