# 要避免的输出（反例片段）

以下均为**错误示范**：生成或润色 `*-article.md` 时应对照排除。

---

## 1. 假对话：双人轮流喊口号

```markdown
Q：这个工具太震撼了，颠覆行业。
A：没错，我也完全同意，这是划时代的产品。
```

**问题**：无信息增量、无分歧、无边界；营销腔（「颠覆」「划时代」）。

---

## 2. 无 `Q：` 线或混用格式

```markdown
**主持人**：你怎么看？
A：很好。
```

**问题**：主持行须统一为 `Q：`；嘉宾行统一为 `A：`。

---

## 3. 把传闻写成定论

```markdown
A：大家都公认这款工具已经彻底取代了初级设计师，薪资结构会立刻重构。
```

**问题**：社区讨论未核实，不得写「公认」「彻底」式断言。

---

## 4. 列表式简报冒充对话长文

```markdown
## Key Points
- 功能 A
- 功能 B
## Background
一段百科介绍
```

**问题**：体裁应是**多轮对话** + 少量 `##` 分章，而非 KeyPoints/Background 分栏长文。

---

## 5. 连续空泛追问

```markdown
Q：你怎么看？
Q：那你觉得呢？
Q：还有别的吗？
```

**问题**：缺少场景、求证、品类边界等具体问法；须轮换 `dialogue-qa-patterns.md` 中的问法类型。

---

## 6. 另存一篇「第二篇口播母本」

**问题**：同一日期只允许**一篇** `data/output/{DATE}-article.md` 作为 TTS 唯一母本；勿平行再写一篇同用途对话稿。

---

## 7. 缺少总结段或参考链接

```markdown
（对话结束，无总结段）
```

**问题**：每篇 `article.md` 必须在结尾有**总结段**（明确判断或待观察条件），以及 **2-5 条参考链接**（官方文档、新闻报道、社区讨论等）。
